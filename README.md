# Kigali City Services & Places Directory (KCS Mobile App)

A Flutter mobile application developed as part of *Individual Assignment 2*.
The app helps Kigali residents locate and manage listings for essential public services and leisure locations (hospitals, police stations, libraries, cafés, parks, etc.).
All data is stored in Firebase Cloud Firestore, and users authenticate via Firebase Authentication.
State management is handled with **Provider** to ensure clean separation between UI and business logic.

---

## 🚀 Features

- **User authentication** with email/password and enforced email verification.
- **User profiles** stored in Firestore and linked to listings.
- **Create, read, update, delete** (CRUD) service/location listings with real-time updates.
- **Search by name** and **filter by category** (Hospital, Police, Library, Restaurant, Café, Park, Tourist Attraction).
- **Detail page** for each listing including full information and an embedded **Google Map** marker.
- **Navigation button** to launch turn-by-turn directions in Google Maps.
- **My Listings** tab showing listings created by the logged-in user with edit/delete actions.
- **Map View** tab displaying all listings as map markers.
- **Settings** screen showing user profile and a toggle for simulated location notifications.
- Bottom navigation bar with Directory, My Listings, Map, and Settings screens.

## 🛠 State Management & Architecture

- **Provider** is used for all shared state.
- `AuthProvider` wraps FirebaseAuth and user profile creation.
- `ListingProvider` manages Firestore listing streams and CRUD operations.
- UI widgets never call Firebase directly; they access providers instead.
- `services/firestore_service.dart` encapsulates Firestore queries.

This architecture satisfies the rubric requirement of a clear service → state → UI flow.

---

## 🧾 Firestore Database Schema

- **users** collection
  - document ID = user UID
  - fields: `uid`, `email`, `displayName`

- **listings** collection
  - `name` (string)
  - `category` (string)
  - `address` (string)
  - `contactNumber` (string)
  - `description` (string)
  - `latitude` (number)
  - `longitude` (number)
  - `createdBy` (user UID)
  - `timestamp` (Firestore Timestamp)

All reads use real-time listeners so that UI updates automatically.

## 🧩 Setup & Running the App

1. **Clone the repository** and open in VS Code.
2. **Install Flutter & Dart** (v3.10+ recommended).
3. **Configure Firebase**
   - Install `flutterfire_cli` and add it to your PATH.
   - Run `flutterfire configure --project=<your-project-id>` in `kcsmobileapp/`.
   - This generates `lib/firebase_options.dart` and adds native config files.

4. **Fetch packages**: `flutter pub get`
5. **Run**: `flutter run` on an emulator or physical device (Android/iOS).
   - The app requires real device/emulator; browser/web is not supported per assignment.

6. **Testing flows**:
   - Sign up with a new email, verify via link, then log in.
   - Create, edit, delete listings and observe Firestore console.
   - Use search/filter, open detail pages, and launch navigation.
